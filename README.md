# Авторедактор журнала

Автоматизация рутинной работы редактора журнала: сортировка статей, сборка в PDF, формирование содержания.

## Возможности

- Сортировка статей по алфавиту (латиница → кириллица)
- Конвертация статей из Word (.docx) в PDF
- Автоматическая нумерация страниц
- Формирование содержания с указанием страниц
- Поддержка титульного листа, вступительных и заключительных страниц
- 4 отступа перед каждой статьёй

## Установка

```bash
pip install -r requirements.txt
```

## Использование

### Базовый запуск

```bash
python autoeditor.py ./articles ./output
```

### С дополнительными страницами

```bash
python autoeditor.py ./articles ./output \
    --title title.pdf \
    --first pages_1_2.pdf \
    --end last_pages.pdf \
    -o my_journal.pdf
```

### Параметры

| Параметр | Описание |
|----------|----------|
| `articles_dir` | Папка со статьями в формате .docx |
| `output_dir` | Папка для выходных файлов |
| `-o, --output` | Имя выходного файла (по умолчанию: journal.pdf) |
| `--title` | PDF титульного листа |
| `--first` | PDF с 1-2 страницами журнала |
| `--end` | PDF с последними страницами |

## Структура проекта

```
articles/           # Статьи авторов (.docx)
├── Article_A.docx
├── Статья_Б.docx
└── ...

output/             # Результат
└── journal.pdf     # Готовый журнал
```

## Формат статей

- Формат: Microsoft Word (.docx)
- Первый абзац документа используется как название статьи
- Название влияет на сортировку и отображается в содержании

## Порядок сборки журнала

1. Титульный лист (если указан)
2. Страницы 1-2 (если указаны)
3. Статьи (отсортированы по алфавиту, латиница → кириллица)
4. Содержание (с нумерацией страниц)
5. Последние страницы (если указаны)

## Пример использования из кода

```python
from autoeditor import JournalEditor

editor = JournalEditor(
    articles_dir='./articles',
    output_dir='./output',
    title_page='title.pdf',
    first_pages='intro.pdf',
    end_pages='outro.pdf'
)

editor.build_journal('my_journal.pdf')
```
